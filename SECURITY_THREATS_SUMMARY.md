## Positive Controls Already in Place

- **Auth**: JWT verified via Supabase in `requireAuth` and WebSocket auth.
- **Session/access**: Most session-scoped routes (sessions, media, decisions, injects publish, etc.) check trainer ownership or participant membership.
- **Sensitive data**: Pino redacts `authorization`, `*.password`, `*.token`, `*.apiKey`, `*.secret`.
- **Rate limiting**: General API limiter and stricter limiter for `/api/join`; body size limits on API and join.
- **Security headers**: Helmet with CSP in production; CORS origin allowlist.
- **Secrets**: `.env` and `.env.*` in `.gitignore`; no `.env` or `.env.example` with real secrets found in the repo.
- **Notifications**: Mark-as-read scoped by `user_id` in `notificationService`.
- **Decisions**: Execute and approve flows check ownership/approval state.
