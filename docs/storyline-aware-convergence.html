The problem

The storyline is generated, then ignored

Step 3 produces 20-30 storyline injects. Step 4 (Convergence) is supposed to design shared chaos + escalation gates as consequences of that storyline — but the wizard hands it an empty object.

Step 3 · Storyline20-30 injects
✗
Step 4 · Convergenceshared injects + gates
frontend/src/pages/SocialCrisisWizard.tsx · generateConvergence
body: JSON.stringify({ crisis_type, country, personas, fact_sheet, team_storylines: {}, // ← storyline thrown away })

The endpoint and generator already support a storyline summary — it just never receives anything, so the prompt slot is blank.

The fix

Feed the storyline in — one line

No backend or schema changes. The convergence endpoint already passes team_storylines straight to generateConvergenceLayer, which builds a summary from it.

SocialCrisisWizard.tsx
- team_storylines: {}, + team_storylines: storylineInjects.length > 0 ? { storyline: storylineInjects } : {},
The trap

The stale-closure gotcha (must-fix)

This is the same class of bug we just fixed in compile. generateConvergence is a useCallback whose deps don't include storylineInjects — which is set in Step 3, after those deps last changed.

without the dep

The memoized closure captures storylineInjects = [] from Step 2. So even with Change 1, convergence would still get an empty storyline. Silent.

with the dep
}, [crisisDescription, country, communities, personas, factSheet, storylineInjects]);

The callback re-memoizes when the storyline lands, so it sends the real injects.

How it works

What the AI sees, before vs after

The convergence prompt has a "storylines already created" slot. Filling it lets the model anchor gates to real beats (e.g. an unanswered T+15 leak triggers a T+20 escalation).

→ gates invented in a vacuum; may not match what players actually face.
After

T+8 Whistleblower thread goes viral
T+15 Regulator requests statement
T+30 Boycott hashtag trends …
→ convergence gates fire as organic consequences of these beats.
Why it's safe
  • generateConvergenceLayer only uses teamStorylines to build the summary string — the single storyline key has no other side effects.
  • Timing is correct: convergence runs in Step 4, after Step 3, so the storyline is populated.
  • No backend/schema change — team_storylines is already z.record(string, unknown).
