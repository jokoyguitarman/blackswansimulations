The big picture

One spine, six stages

A trainer authors a scenario in the wizard. Compile freezes it into the database. At runtime a 30-second scheduler turns the script into living news/posts/emails on player phones, scores every action across four dimensions, and feeds it all into an after-action review.

Stage 1

Authoring — the 7-step wizard

The current path is unified (no player teams — everyone shares one feed). Each step calls an endpoint on socialCrisisWarroom.ts and streams progress. Generation fires as the trainer leaves the previous step.

1

Scenario Setup

/upload-brand-logo · /upload-document

Free-text crisis description, country, org name, optional brand logo, and document upload (PDF/DOCX/TXT → text appended into the crisis context). No generation yet.

2

Characters & Facts

POST /generate-npcs (async job + poll)

The character bible + ground truth of the sim.

~18–20 key NPCs ~80 background NPCs fact_sheet · confirmed_facts unconfirmed_claims (+ truth, spread_by) affected_communities
3

Storyline

POST /generate-storyline (NDJSON stream)

20–30 injects forming the escalating arc (shock → outrage → blame → demand → resolution). Every inject carries a delivery_config.app.

Convergence + Org Page

POST /generate-convergence → POST /generate-org-page

Convergence produces the narrative and the scoring scaffold; the org-page call builds the brand's social presence.

title / description / briefing objectives shared_injects convergence_gates dimension_labels FB + X identity + logo 10–20 branded_history posts
5

Sentiment Research

POST /research-sentiment (NDJSON stream)

A PublicSentimentProfile: analogous real cases, a 5-phase reaction arc, platform behaviors, demographic splits, counter-narrative effectiveness.

used only in the wizard UI — never at runtime

POST /research-general → buildSOPFromResearch()

Research yields ResearchGuidelines (per-team + group-wide). The SOP itself is mostly hardcoded (10 fixed steps + content guidelines; only the publish-step timing is overridden).

Review & Compile

POST /compile (async job + poll)

Generates strategy windows, assembles the payload, persists everything, and pre-generates NPC post images in the background.

Stage 2

What compile actually writes

Two destinations: a single JSON "world model" in scenarios.initial_state, and relational rows. Tagged tiles below show which generated fields are live vs. dead at runtime.

scenarios.initial_state JSONB

npc_personas fact_sheet research_guidelines affected_communities dimension_labels org_name org_page (+ branded_history) country sentiment_curve sentiment_profile facebook_groups strategic_benchmarks*

*strategic_benchmarks is a type slot only — the generator never populates it (see §7).

Relational rows

  • scenario_injects time / condition / decision injects — each with delivery_config
  • scenario_objectives scored objectives (weight + success criteria)
  • sop_definitions steps, time limits, content guidelines
  • scenario_teams empty in unified mode

Injects are split by assemblePayload: those with trigger_time_minutes → time; convergence gates + strategy windows → condition; trigger_condition → decision always empty

Stage 3

Runtime — the 30-second heartbeat

One InjectSchedulerService interval drives everything. For each in-progress social session it walks five steps, then the feed & ambient engines turn the frozen script into living content.

30sper session tick
1
Time injects

Due injects → shouldCancelSocialInject AI gate (reads player actions, sentiment, pending count, guidelines) → publish.

2
computeSocialState

Recompute the 4 dimensions from posts + actions + grades.

3
Condition injects

conditionEvaluator checks appear/cancel keys against social_state → AI gate → publish.

4
checkResponseDeadlines

Unanswered requires_response posts past their deadline apply a −10 sentiment hit.

5
runEngagementTick → generateAmbientPosts

Engagement math (impressions/likes) and all ambient generation live here.

Feed engine — routeInjectToApp by delivery_config.app

When an inject publishes, it's materialized into the right table(s) and broadcast over WebSocket to the player phones.

news inject→
sim_news_articles row created + dual cross-post to social_posts on X & Facebook (carrying shared_article_id).
Broadcasts news_article.published + social_post.created.
social_feed inject→
One social_posts row (sentiment from content_flags, virality, engagement seed). If spawn_replies>0, async AI generates 1–8 NPC reply tweets.
email inject→
sim_emails row (parses Subject:, sanitized email_category — verified_facts, legal_advisory, approval_chain…). Broadcasts sim_email.received.
group_chat inject→
chat_messages into the channel matching channel_type (tips, leaks, whistleblowers).
phone_call inject→
WebSocket phone_call.incoming only — no DB row, and there is no phone app to receive it. orphaned

Stage 5

State & scoring — the four dimensions

Every tick, computeSocialState recomputes four 0–100 dimensions from posts, actions and AI content grades, then a weighted composite. Bars animate to illustrative values when in view.

Public Trust0
base 50 + grade accuracy + 3×fact-checks − unaddressed misinfo
Community Safety0
base 40 + 15(leader contacted) + sensitivity − fear posts − 10(rally)
Narrative Control (40% weight)0
20 + min(impression-ratio,2)×30 + 10(official) − 0.5×weighted hate
Escalation Risk0
20 + 12×rally + 8×violence-inciting − reporting actions

composite = 0.2·trust + 0.2·safety + 0.4·narrative + 0.2·(100−escalation), then ×1.1 if strategic_ratio>0.4 or ×0.85 if <0.2. Designed-NPC harmful posts weigh 3×.

Condition DSL conditionEvaluatorService

Condition injects appear when { all: [...] } or { threshold: N, conditions: [...] } is met, and die when any conditions_to_cancel key is true. Keys read current_state.social_state (sentiment thresholds, unaddressed counts, SOP steps, semantic signals, action patterns).

AI content grader on every player post

Grades accuracy / tone / cultural_sensitivity / persuasiveness + 8 boolean signals → written to social_posts.sop_compliance_score. drops fact-sheet claims it only receives confirmed_facts, never unconfirmed_claims.

Watchdog & disputes

statementWatchdog (120s) scans player statements for contradictions → challenge cascade. contentDispute lets players file takedowns → AI judge rules uphold / correct / reject → retract/correct/remove articles & posts.

Stage 7

Gaps, dead code & dropped wiring

Verified by four parallel code-audit agents. The core spine is solid — these are the branches that are generated-but-unused, sent-but-dropped, or defined-but-never-triggered.

sentiment_curve dead data

Written to initial_state, never read. All dimension math uses hardcoded baselines (50/40/20/65).

strategic_benchmarks dead data

generateStrategicBenchmarks is never called; field never populated → AAR doctrine compliance is always empty.

facebook_groups dead data

Curated groups never read; runtime groups are AI-improvised by initializeGroups.

sentiment_profile (runtime) dead data

Used only in the wizard UI — never at runtime or in scoring.

org_name dropped

Accepted by every endpoint but only forwarded to org-page generation — ignored for NPCs, storyline, convergence, sentiment, best-practices.

fact-sheet claims → grader dropped

Grader only gets confirmed_facts; never unconfirmed_claims/specific_claims, so it can't verify against known-false claims.

decision_injects orphan

Always empty — unified storyline never sets trigger_condition.

phone calls & draft approval orphan

Action types (call_answered, draft_approved…) never emitted; no phone app, no approval workflow. approval_chain is persisted but never enforced.

Architectural caveats
  • gateEvaluationService is effectively a no-op for social — no scenario_gates rows; convergence gates run through conditionEvaluator instead.
  • Consequence edge-state is an in-memory Map — resets on restart, not multi-instance safe.
  • A fresh non-MBS session shows an empty News app if the AI key is unset (migration 182 only seeds the hardcoded MBS scenario).
  • Orphaned exports: sendNPCDirectMessage, resetTickCounter, doctrine-cache benchmark fns — never called.
