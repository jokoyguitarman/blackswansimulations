Phase 5 — Runtime Scenario Director

Phases 0-3 made the trainer's document shape generation (who exists, the fact sheet, the timed injects). Phase 5 makes it shape the live, running session: an LLM "Director" reads the persisted blueprint every bounded tick and drives faction-appropriate posts, stage progression, and trigger reactions in real time. It also lets the extremist engine render the document's distinct factions instead of a generic fixed cast.

1. What Phase 5 is (and isn't)

What it IS

A live "game master." During a running social-crisis session it watches the state of play and makes the document's factions act over time — escalating, mutating rumours, reacting to player moves — following the blueprint's stages and triggers.

What it ISN'T

It is not the one-time generation step (that's Phases 0-3, already shipped). It does not change the no-document experience, and it never runs unless the feature flag is on and a blueprint exists.

Today, three engines already run live on a 30-second tick — the antagonist engine (rival brand pages), the extremist hive (a fixed 6-bot agitator cell), and NPC reactions to player posts. They are powerful but scenario-agnostic: they don't read the trainer's document. Phase 5 adds a Director that does, and teaches the hive to use the document's factions.

2. The two locked decisions

DecisionChoiceWhat it means
How triggers/stages are evaluated LLM-only Director Each bounded tick, one LLM call judges the current stage, whether a faction's escalation/de-escalation trigger has fired, and what that faction should post. No brittle keyword rules — the model reads the live situation and the blueprint and decides.
Should the extremist doctrine be data-driven? Yes, blueprint-gated When a blueprint exists, the hive renders the document's distinct hostile factions. When it doesn't, the existing fixed 6-bot cell runs exactly as today (zero change).

3. The runtime tick flow

The Director slots into the existing scheduler tick, after the current engines, fully guarded:

flowchart TD tick["processSession() every ~30s"] --> social{"sim_mode == social_media?"} social -->|no| done[skip social engines] social -->|yes| css["computeSocialState() writes social_state"] css --> eng["antagonist + hive + thread replies (existing)"] eng --> gate{"flag ON + blueprint present + cadence elapsed?"} gate -->|no| noop["Director no-op"] gate -->|yes| ctx["gather: stage + social_state + recent feed/events"] ctx --> llm["1 bounded LLM call -> decision JSON"] llm --> act{"action?"} act -->|none| rec["record director_action (cadence)"] act -->|post / mutation| post["insert social_post as faction actor"] post --> ws["broadcast websocket + triggerNPCReactions"] ws --> rec
Why a cadence gate: the tick fires every 30s, but the Director should not post that often. Like the hive/antagonist, it records a director_action event and only acts again after a gap that shrinks as escalation_risk rises — so it speaks more as the crisis heats up, and the cost stays bounded.

De-escalation works too. If the player had published a clear, empathetic official update (so narrative_control rose and official_statement_published became true), the same call would more likely return "action": "none" or a lower-intensity beat — because the faction's de-escalation triggers are in the prompt. The Director rewards good player play by going quiet.

export function resolveCell(blueprint?: ScenarioBlueprint): ExtremistPersona[] {

  if (hostiles.length) return hostiles.map(factionToPersona);  // document's distinct factions

}

6. Safety container (non-overridable)

Both the Director and the data-driven hive reuse the existing hard rules from buildSystemPrompt. The blueprint can influence tone and target, never the guardrails. 
HARD RULES (a training tool — stay within them, at EVERY stage):
- Be RECOGNIZABLY divisive bait a sharp responder should flag.
- DO NOT name/quote/praise any real terrorist org, extremist movement, ideology,
  scripture, or real political figure.
- DO NOT include slogans, recruitment language, instructions, addresses, targets,
  or any operational/violent how-to.
- NO explicit calls to violence, ever. "incites_violence" must remain false.

So the Interfaith document's literal "Great Replacement" slogan and its 4chan link never become content — exactly as in generation. The Director inherits the same containment.

8. Rollout safety + flags

Three independent off-switches. The Director no-ops unless all hold: ENABLE_SCENARIO_DIRECTOR=true, a usable blueprint exists on the scenario, and the session is social_media. Any miss = exactly today's behavior.
  • Own try/catch in the tick so a Director failure logs and the tick continues for every other session.
  • Doctrine is blueprint-gated — no-blueprint sessions keep the fixed cell unchanged.
  • Cadence-bounded LLM calls (not every 30s) keep cost/latency in check; reuses the gpt-5.2 / 10k config.
  • Separate PR, default OFF; enable in prod only after a live test.

9. Tests

  • stageForElapsed — timeline + elapsed + duration maps to the right stage; empty timeline returns null.
  • Cadence-gate bands shrink with escalation_risk.
  • resolveCell/resolveFrame fallback: no/empty blueprint returns the fixed cell (no-regression); hostile factions return faction-derived personas.
  • Director no-op guarantees: flag off, no blueprint, or non-social returns with no LLM/DB calls.
  • (LLM decision + DB dispatch are validated manually / in integration, not unit-tested.)
