Frontend Enhancement

Let Trainers Preview Deterioration Over Time

The timeline slider already exists, but it only colors studs. Three targeted changes will make spawn pins appear/disappear as the trainer scrubs through time.

The Problem

Everything Is Shown at Once

Right now, the SceneCanvasView renders all hazard pins simultaneously regardless of status or timing. The timeline slider controls fire-spread stud coloring but doesn't affect pin visibility. A trainer can't tell which hazards appear at T+5 vs T+20.

Current Behavior
  • -- All 11 hazard pins visible at every time step
  • -- Delayed spawn pins (fire spread) look identical to initial hazards
  • -- Timeline slider only changes stud grid coloring
  • -- Hazard info popup has no deterioration timeline
  • -- No visual link between fire-spread studs and spawn pins
Target Behavior
  • -- Only pins with appears_at_minutes ≤ slider time are visible
  • -- Delayed pins appear as ghost/dashed before their time, solid when reached
  • -- Slider controls both stud coloring AND pin visibility
  • -- Clicking a pin shows its deterioration timeline in the info panel
  • -- The stud where a spawn pin lands gets a cyan highlight
