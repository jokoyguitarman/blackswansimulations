The critical constraint

State won't update mid-chain — thread the data

Today each generator reads the previous step's output from React state, which only works because steps are separated by user clicks. In one synchronous chain, state hasn't updated yet between calls.

the trap

generateStoryline reads personas/factSheet from state (SocialCrisisWizard.tsx:566-567). Call it right after generateNPCs() in a loop and those are still empty — React batches state updates.

the fix

Refactor each generator to accept explicit inputs and return its result (still call setState for the UI). generateAll passes each return into the next call.

Functions to refactor (return value + optional args)
  • generateNPCs (:475) → returns { personas, factSheet, communities }
  • generateStoryline(personas, factSheet) (:548) → returns storylineInjects
  • generateConvergence(personas, factSheet, storyline) (:608) → returns narrative/objectives/shared/gates/labels
  • generateOrgPage(roster) (:703) → sends allies/competitors/auto_antagonist; returns orgPage
The engine

generateAll() orchestrator

One function runs the chain in order, tracks the live stage, stops on failure (with retry), and auto-advances to compile on success.

const generateAll = async () => { const npc = await generateNPCs(); // { personas, factSheet, communities } if (!npc) return setBuildError('characters'); const story = await generateStoryline(npc.personas, npc.factSheet); if (!story) return setBuildError('storyline'); const conv = await generateConvergence(npc.personas, npc.factSheet, story); if (!conv) return setBuildError('convergence'); await generateOrgPage({ allyEntries, competitorEntries, autoAntagonist }); await saveDraftState(7); setStep(7); // auto-advance to compile };

Track buildStage (characters | storyline | convergence | pages | done) and buildError to drive the progress UI and the retry button.

Step 1

Setup absorbs the roster

Competitor/ally inputs move up from the old Step 4 so the chained org-page generation receives them — named competitors get full AI identities + branded history in the same pass.

New Setup state

  • allyEntries[] — protagonist ally names (+ optional handles)
  • competitorEntries[] — antagonist competitor names (1–10)
  • autoAntagonist — default true (invent a rival if none named)

UI change

Move the two-panel ally/competitor UI from renderStep4 into renderStep1. Replace the old addExtraPage/getAllies/getAntagonists (which mutate orgPage.orgs) with simple name lists — orgPage doesn't exist yet at Setup time.

Step 2

Building: progress-only, auto-advance

A single renderBuilding() shows a per-stage checklist driven by buildStage, a spinner, and a retry on buildError. On success it calls setStep(7) automatically.

Auto-advance

On the final stage completing, generateAll saves the draft and calls setStep(7) — the user lands on Compile without an extra click.

Retry on failure

If a stage fails, the chain stops (no auto-advance) and the failed stage shows a retry — one bad AI call doesn't force a full restart.

No review/edit

Per the chosen design, Step 2 is purely progress. The old renderStep2/3/4 bodies are replaced by renderBuilding().
