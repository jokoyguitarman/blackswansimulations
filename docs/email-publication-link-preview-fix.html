Fix: Email-Published Articles → Link Preview Integration

When a player emails a journalist NPC and provides enough information, the journalist publishes a news article. Currently, the resulting social media post is a plain text post with no link to the article. This fix connects it to the link preview system.

The Problem
Gap

Email-Published Articles Are Disconnected from Link Previews

The journalist NPC publication flow in npcEmailReplyService.ts (lines 485–554) has three issues:

What's MissingConsequence
No shared_article_id on the social post Link preview card doesn't render — it's just a plain text post
No content_flags.shared_article metadata Even if shared_article_id was set, the card has no headline/outlet/snippet to display
No news_article.published WebSocket broadcast The News App never receives the article in real-time — player must refresh
Only posts to X/Twitter, not Facebook Fakebook players never see the journalist's story

server/services/npcEmailReplyService.ts — lines 485–554

What Changes
Fix

Single file: npcEmailReplyService.ts (lines 496–545)

Step 1: After the article is inserted successfully (line 507), add the WebSocket broadcast so the News App receives it in real-time.

// After article insert succeeds, broadcast to News App
getWebSocketService().broadcastToSession(sessionId, {
  type: 'news_article.published',
  data: { article },
  timestamp: new Date().toISOString(),
});

Step 2: Replace the single-platform plain text social post (lines 517–545) with the dual-platform linked pattern from feedEngineService.ts.

// REMOVE: old plain text X-only post
const socialContent = `BREAKING: ${headline}\n\n${articleBody.substring(0, 200)}...\n\nFull story available.`;
const { data: post, error: postError } = await supabaseAdmin
  .from('social_posts')
  .insert({ ...plain text fields, no shared_article_id... })

// REPLACE WITH: dual-platform linked posts
const snippet = articleBody.substring(0, 150);
const categoryLabel = category === 'breaking' ? 'BREAKING' : category.toUpperCase();
const sharedArticleFlags = {
  shared_article: { id: article.id, headline, outlet_name: outletName, snippet, category }
};
const platforms = [
  { platform: 'x_twitter', content: `${categoryLabel}: ${headline}\n\nnews.sim/...` },
  { platform: 'facebook', content: `📰 ${headline}\n\n"${snippet}..."\n\n— ${outletName}` },
];
for (const { platform, content } of platforms) {
  // insert with shared_article_id + content_flags
  // broadcast social_post.created
}

No other files change. The LinkPreviewCard rendering and click-to-navigate logic in SocialFeedApp, FacebookFeedApp, and NewsApp are already wired up from the previous commit.
