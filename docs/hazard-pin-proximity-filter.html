Why Hazard Pins Appear Outside the Building

DB hazard pins are rendered at their raw lat/lng coordinates without checking if they're near the building polygon. Pins from zone-based spawning or exterior studs show up far from the scene.

Root Cause

Three Reasons Pins Land Outside the Building

1

Exterior Padding Studs (150m)

buildingStudService.ts generates studs with EXTERIOR_PADDING_M = 150 beyond the building footprint. The environmental simulation runs fire spread across ALL studs including these distant exterior ones. When a fire-spread spawn pin lands on an exterior stud, it gets coordinates far from the building.

2

Zone-Based Pin Spawning

The old deterioration system (hazardDeteriorationService) uses placeInsideZoneType('hot') which picks a random point inside the hot zone polygon. Hot zones can extend well beyond the building footprint, placing pins in streets, parks, or neighboring blocks.

3

No Rendering Filter

The new visibleDbHazards filter only checks appears_at_minutes vs the timeline slider. It does NOT check whether the pin's lat/lng is near the building. Every DB hazard for this scenario gets a Leaflet marker regardless of distance.

The Fix

Bounding Box Proximity Filter

Compute a padded bounding box from the building polygon. Only render pins inside this box as Leaflet markers. Off-site pins still exist in the DB but are simply not rendered on the scene canvas.

Single file change: SceneCanvasView.tsx

Step 1: Compute building bounding box
tsxconst buildingBounds = useMemo(() => { const poly = sceneConfig?.building_polygon; if (!poly || poly.length < 3) return null; const lats = poly.map(p => p[0]); const lngs = poly.map(p => p[1]); const PAD = 0.001; // ~111 meters in all directions return { minLat: Math.min(...lats) - PAD, maxLat: Math.max(...lats) + PAD, minLng: Math.min(...lngs) - PAD, maxLng: Math.max(...lngs) + PAD, }; }, [sceneConfig]);
Step 2: Add proximity check to the partition
tsxconst { visibleDbHazards, futureDbHazards } = useMemo(() => { const isNearBuilding = (h: Record<string, unknown>) => { if (!buildingBounds) return true; const lat = h.location_lat as number; const lng = h.location_lng as number; return ( lat >= buildingBounds.minLat && lat <= buildingBounds.maxLat && lng >= buildingBounds.minLng && lng <= buildingBounds.maxLng ); }; const visible = dbHazards.filter(h => (h.appears_at_minutes as number ?? 0) <= currentTimelineMinutes && isNearBuilding(h) // NEW ); const future = dbHazards.filter(h => (h.appears_at_minutes as number ?? 0) > currentTimelineMinutes && h.status === 'delayed' && isNearBuilding(h) // NEW ); return { visibleDbHazards: visible, futureDbHazards: future }; }, [dbHazards, currentTimelineMinutes, buildingBounds]);

What this does NOT do

  • -- Does not delete any data from the database
  • -- Does not affect the live session deterioration service
  • -- Does not change how pins are generated at compile time
  • -- Only affects rendering in the SceneCanvasView trainer preview
