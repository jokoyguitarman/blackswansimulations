01 · The concept

Who the hive is, and how it differs from what exists

The simulation already has three "casts" of voices. The Extremist Hive is a fourth: a small, fixed, ideologically-driven cell that is present in every session regardless of what the trainer authored. It does not shout constantly — it watches, waits, and exploits.

Existing casts today

  • Scenario NPCs — AI-generated per crisis, stored in initial_state.npc_personas. Vary every session.
  • Antagonist org pages — rival brands tied to scenario org config, run by runAntagonistEngine. Only exist if authored.
  • Ambient / background NPCs — improvised crowd noise each tick.

The new cast: Extremist Hive

  • Fixed & code-owned — a hardcoded roster, identical archetypes every session.
  • Scenario-agnostic — appears in every social-crisis session, no wizard step.
  • Opportunistic — lurks; only posts when it detects a seam to widen.
  • High-impact — treated as a designed NPC, so its harmful posts weigh 3x in scoring.

The core behavior in one sentence

Each 30-second tick, the hive reads the live battlefield (recent posts, current social_state, the fact sheet's unverified claims, demographic fault lines). If — and only if — it detects an exploitable opening and its cadence cap has elapsed, one cell member posts an adaptive, divisive message flagged with content_flags; other NPCs then pile on, and the existing state engine converts that into a measurable swing in escalation, narrative control, and public consensus.

03 · Disconnect → Connect

What is missing today, and the single seam that joins it

Almost everything the hive needs already exists and is wired. The only true gap is the actor itself: there is no persistent, scenario-agnostic extremist faction, and nothing calls one from the tick loop.

Gap 1 — No persistent hostile faction

The only adaptive hostile actor (runAntagonistEngine) requires authored antagonist org pages (sim_org_pages with role = 'antagonist'). Sessions without them have no smart hostile voice beyond improvised ambient noise.

Gap 2 — No tick wiring

The social-media branch of the scheduler calls engagement and antagonist engines only. Nothing invokes an extremist engine.

Gap 3 — No doctrine / opportunism logic

There is no "watch and wait for an opening" gate anywhere; the antagonist posts on a simple escalation-scaled timer.

Connect 1 — Fixed roster in code

A hardcoded cell in extremistDoctrine.ts, registered each session, independent of the wizard. Their handles are flagged as designed NPCs so the existing 3x weighting applies.

Connect 2 — One line in the loop

Add await runExtremistHive(session.id, elapsedMinutes) right after the antagonist call in the existing sim_mode === 'social_media' block.

Connect 3 — Opportunity gate

A new gate reads social_state + recent posts + demographics and only fires on a detected seam, with a cadence cap recorded in session_events.

Already connected — no work needed

export async function runExtremistHive(
  sessionId: string, elapsedMinutes: number,
): Promise<void> {
  if (!env.openAiApiKey) return;

  // 1. Load live battlefield
  const { social_state, initial_state } = await loadContext(sessionId);
  const recentPosts = await loadRecentPosts(sessionId);

  // 2. OPPORTUNITY GATE - lurk unless there is a seam to widen + cadence ok
  const opening = detectOpening(social_state, recentPosts, demographics);
  if (!opening) return;
  if (!cadenceElapsed(sessionId, elapsedMinutes, social_state)) return;

  // 3. Pick member + move, build doctrine prompt, call AI
  const member = pickCellMember(recentPosts);
  const result = await callAI(buildSystemPrompt(ctx), userPrompt);
  if (!result?.content) return;

  // 4. Insert as harmful post (flags drive the swing), broadcast, pile-on
  const post = await insertHivePost(sessionId, member, result);
  getWebSocketService().broadcastToSession(sessionId, { type: 'social_post.created', ... });
  void triggerNPCReactions(sessionId, post);

  // 5. Record cadence event
  await supabaseAdmin.from('session_events').insert({ event_type: 'extremist_post', ... });
}

05 · The Opportunity Gate

What makes them extremists, not just another antagonist

The antagonist posts on a timer. The hive lurks. The gate is the "special kind of mind" you described — it stays silent until the feed offers a seam worth widening, then strikes with restraint to stay believable.

Exploitable signals (any one can open the gate)

  • Fresh unaddressed harmful post — an existing inflammatory post it can amplify or twist (unaddressed_hate_count > 0).
  • Demographic fault line — players clustered by demographics create groups to set against each other.
  • Rising escalation — escalation_risk trending up.
  • Collapsing narrative — narrative_control low or falling.
  • Conspicuous silence — no official_statement_published past a time threshold.
  • A real grievance in recent posts to hijack.

Restraint rules (keep it credible)

  • Cadence cap — a requiredGapMinutes(escalationRisk)-style gate, checked against the last extremist_post in session_events. Rarer than the antagonist.
  • One voice at a time — rotate cell members; never flood.
  • Plausible deniability — never state the hidden ideology outright; ride existing grievances.
  • No opening, no post — if nothing is exploitable, stay completely silent this tick.
06 · Emotion & Consensus

How a hive post swings the public — using zero new plumbing

A hive post is just a social_posts row with the right content_flags. Those flags are exactly what computeSocialState already scans every tick, so the swing is automatic.

Flags the hive sets

is_inflammatory, is_harmful_narrative, incites_violence, is_organized_pressure, is_misinformation — the exact keys the state engine reads.

Why it hits hard

Hive handles are designed NPCs, so unaddressed harmful posts weigh 3x in the penalty math (socialStateUpdaterService.ts ~227-238).

Knock-on cascade

Pushing narrative_control below 20 or activating a rally triggers narrative_collapsing / rally_gaining_traction consequence injects — the divide compounds.

07 · The Brain

What the classified documents will define

Everything sensitive lives in one isolated, data-only module: extremistDoctrine.ts. The engine is generic; the documents shape behavior without touching wiring.

Distilled into four structures

  • Persona archetypes → EXTREMIST_CELL roster (cover identity + hidden driver + voice).
  • Tactic catalog → EXTREMIST_MOVES (the manipulation playbook).
  • Trigger conditions → opportunity-signal definitions for the gate.
  • Guardrails → restraint + in-character rules baked into the system prompt.

Handling

  • Documents are treated as design source; I distill them into a behavior model, not reproduce them verbatim.
  • The hidden ideology never appears literally in any generated post — only its effect (division) does.
  • Content stays bounded for a training tool; guardrails prevent it from becoming an operational how-to.
Status: awaiting documents

The doctrine module ships as a structured placeholder. Once I study the material, I populate the four structures above — no other file needs to change for the brain to come online.

09 · Expectations

What "done" looks like

Acceptance criteria, risks, and what is explicitly out of scope.

Acceptance criteria

  • Every social-crisis session features the cell, with no wizard/authoring step.
  • The hive stays silent when there is no opening; it does not flood the feed.
  • When it strikes, the post is in-character, divisive, and carries appropriate content_flags.
  • Hive posts measurably move escalation_risk / narrative_control (verifiable in social_state).
  • NPCs react to hive posts; cadence prevents spam; events are logged for AAR.
  • If openAiApiKey is unset, the engine no-ops cleanly (no errors).

Risks & mitigations

  • Over-posting → strict opportunity gate + cadence cap, rarer than antagonist.
  • AI cost → gate short-circuits before any AI call on most ticks.
  • Cadence state lost on restart → cadence read from session_events (DB), not in-memory.
  • Content too explicit → guardrails in system prompt; never literal ideology.
  • Handle weighting → verify hive handles hit the designed-NPC 3x path.

Out of scope

No new DB tables / migrations No new social_state dimension No wizard / authoring UI No frontend changes (posts render through existing feed)
