Comment Avatar Logo Fix

The organization page logo (e.g. Marina Bay Sands) shows correctly on its posts, but every comment made by the page falls back to the initial letter "M" - for all viewers. This documents the concept, the change, and the result.

The concept Frontend

Avatars are drawn by two different pieces of code: one for the post header, one for each comment. The recent fix taught the post avatar to use the page logo, but the comment avatar was never updated - it still always draws a colored circle with the first letter of the name.

official_account author → Post avatar: looks up orgPageLogos[handle] ✓ vs Comment avatar: always letter circle ✗
Why it happens: the comment/reply render path has no author_type === 'official_account' branch, so it never reaches for the logo. It unconditionally renders getAvatarColor(name) + the first letter. 
// comment/reply avatar today - letter only, no logo branch
<div style={{ backgroundColor: getAvatarColor(reply.author_display_name) }}>
  {reply.author_display_name.charAt(0).toUpperCase()}
</div>

The change Frontend

Fix: give the comment/reply avatar the same logic as the post avatar - if the author is an official_account and a logo exists for its handle, render the logo image; otherwise keep the letter circle. 
- <div style={{ backgroundColor: getAvatarColor(reply.author_display_name) }}>
-   {reply.author_display_name.charAt(0).toUpperCase()}
- </div>
+ {reply.author_type === 'official_account' && orgPageLogos[reply.author_handle] ? (
+   <img src={orgPageLogos[reply.author_handle]} className="...rounded-full object-cover" />
+ ) : (
+   <div style={{ backgroundColor: getAvatarColor(reply.author_display_name) }}>
+     {reply.author_display_name.charAt(0).toUpperCase()}
+   </div>
+ )}

Applied in three comment/reply locations

FacebookFeedApp.tsxInline comment/reply avatars under feed posts SocialFeedApp.tsxZ (Twitter) reply avatars in the thread view OrgPageView.tsxComment list avatars - also needs the all-pages logo map added (it currently only loads its own page)
orgPageLogos = page_handle → page_logo_url (already loaded in the feeds)
