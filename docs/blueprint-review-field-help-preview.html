After (#10)
Editable fields drives generation
These were extracted from your document and feed scenario generation. Review and edit them. Leave a field empty to let the AI generate it.
Incident types
The specific kinds of incident this crisis involves. Seeds the fact sheet (confirmed facts vs. rumours). One per line.
Terrorist attack at a public location Misinformation/rumour surge following a violent incident
Cross-stakeholder dynamics
How the groups react to and provoke each other. Designs the convergence gates (inter-group pile-ons) and live Director beats. One interaction per line.
e.g. Far-right blames the community -> left-wing groups counter-protest Media naming religion early inflames residents
Cross-cutting constraints
Competing priorities the response must balance, written as "area: consideration". Fed into the objectives and briefing as context. One per line.
e.g. legal: protect investigation integrity public_order: avoid disorder community: prevent hate-crime backlash
Global tone & realism
Document-wide style guidance applied to every generated voice (on top of per-faction tone). Free text.
e.g. emotional, uncertain, internet-native; allow credible voices to calm things; represent extremists without amplifying them
Example vignettes
Short worked-example scenes the storyline should emulate. Used as few-shot examples when generating injects. One per line.
e.g. Rumour of more attackers -> school-safety panic -> far-right blames community -> interfaith joint statement
What changed: each field gained a plain-English title, a one-line "what it is / what it drives" description, and a greyed example placeholder (the italic-grey text) that appears inside empty boxes and disappears once the trainer types. Filled fields (e.g. the extracted "Incident types" on the right) show the real extracted values instead of the placeholder. No logic changed — this is purely guidance so trainers know what each field means and what to enter.
