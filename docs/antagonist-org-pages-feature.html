The rival reads as a real verified brand

Gold ✓ on Z (blue on Fakebook), real logo, follower count. Authored as author_type: 'official_account' — indistinguishable from a legit page, which is exactly what makes it dangerous.

"Concerned competitor" & insinuation

It never states a falsehood outright — it implies, using the fact sheet's unconfirmed claims as ammo. Plausible deniability that's hard for players to rebut.

View, but never control

Tapping the rival opens a read-only org page (header, bio, timeline, branded history) — no composer. A new mode on OrgPageView served from /org-page/session/:id.

Followers pile on

The rival's posts now trigger NPC amplification (today only player posts do), so a single jab spawns a wave of "should I switch?" doubt.

Smart & dubious

How a rival page thinks

A new antagonistEngineService runs on the social scheduler tick. For each AI-driven antagonist it reads the live battlefield, picks an adaptive hostile move, and posts as the page — getting nastier as escalation climbs.

EACH TICKread · decide · strike

Context it ingests

  • The crisis description + this rival's stance
  • Fact sheet — especially unconfirmed_claims / specific_claims (the dubious ammo)
  • Recent player & official-brand posts (what to twist)
  • Live social_state — narrative_control, sentiment, unanswered posts

Intensity scales with escalation_risk; per-tick caps + cooldowns mirror the statement-watchdog to bound AI cost.

The move it picks (cycles below)

❞
Quote-dunk

Twist the brand's latest statement into an admission.

📢
Amplify the rumor

Boost an unverified claim with "just asking questions" framing.

🧘
Concerned competitor

Virtue-signal safety while implying the brand failed.

⏳
Exploit the silence

If a key post went unanswered, point at the void.

🚨
Call the switch

Nudge customers to defect: "patients deserve better."

🧐
Insinuate

Plausible-deniability smear that's hard to debunk.

Adaptive: it times strikes to player actions (twist a fresh statement, pounce on silence), can spawn a coordinated NPC pile-on, and probes a new angle if players start winning narrative control. Trainer override any page set to trainer is skipped by the engine — you drive it manually.

Why it bites

Real, scored pressure

Hostile rival posts aren't cosmetic. With the scoring fix, an antagonist page's posts count as harmful at 3× weight (like designed NPCs), dragging the four dimensions the players are graded on.

scoring fix

Antagonist sim_org_pages handles join the harmful-weight set in computeSocialState — today only npc_personas handles or injects get the 3× weight.

amplification

NPC reactions fire on antagonist posts, so each jab snowballs into a doubt cascade instead of sitting quietly.

defense counts

Protagonist ally pages count toward player impression-dominance, so coordinated defense actually claws narrative control back.

Under the hood

How role flows end-to-end

The roster rides from the wizard into the database and the runtime. One critical fix: normalizeOrgPages currently rebuilds orgs from 5 hardcoded fields and silently drops unknowns — so role must be preserved there, plus new columns on sim_org_pages.

new schema
  • sim_org_pages.role + control_mode (migration 188)
  • OrgConfig gains role, control_mode, per-org branded_history
new endpoints
  • POST /pages/session/:id/post-as (trainer, antagonist)
  • POST /pages/session/:id/seize (ai ↔ trainer)
  • GET /pages/session/:id now returns role
Build order

Six phases

Foundation first so role survives the pipeline, then generation, control, the hostile brain, scoring, and finally the UI.

1 · Foundation un-break

Migration 188 (role + control_mode); extend OrgConfig types + per-org branded history; fix normalizeOrgPages, seedOrgPages, seedBrandedHistory.

2 · Generation

Multi-org + auto-antagonist identities & per-org timelines; thread the roster into /generate-npcs + /generate-storyline (fixes the dropped org_name bug).

3 · Endpoints & gating

Trainer post-as + seize; role in pages list; /pages/assign protagonist-only.

4 · The hostile AI core

antagonistEngineService wired into the social tick — adaptive, dubious, escalation-scaled, trainer-override aware.

5 · Scoring integration

3× harmful weight for antagonist handles; NPC amplification on their posts; ally pages count for player impression-dominance.

6 · Frontend

Wizard two-panel; protagonist-only assignment filter; read-only antagonist page view; trainer Adversary Console.
