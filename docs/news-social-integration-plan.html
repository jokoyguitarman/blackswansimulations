Key Design Rule
Design Rule

Hide Factuality & Content Flag Badges from Players

Rationale: If players can see UNVERIFIED / BREAKING badges and content flags on articles and posts, it removes the critical thinking element from the simulation. Players should have to read the news, evaluate the source, check the facts, and decide for themselves whether an article is trustworthy. Trainers and admins still need these indicators for assessment and orchestration.

What this means

There are 3 places in NewsApp.tsx where !article.is_factual renders the UNVERIFIED badge:

  • Article detail view (line ~275) — in the publisher attribution area
  • Hero card (line ~483) — top of the first article card
  • Article list items (line ~547) — in each article row

All three must be gated behind isTrainer.

Critical Bug

Task 1 — Fix Zod Schema for shared_article_id

P0

Problem: The createPostSchema Zod validator does not include shared_article_id or content_flags. Zod's default behavior strips unknown keys, so when a player shares a news article from the News App, the article reference is silently dropped. The post is created but with no link to the article.

File to modify
server/routes/socialMedia.ts — lines 245–267

Current code

const createPostSchema = z.object({
  body: z.object({
    session_id: z.string().uuid(),
    content: z.string().min(1).max(2000),
    reply_to_post_id: z.string().uuid().optional(),
    image_prompt: z.string().max(500).optional(),
    media_url: z.string().url().optional(),
    platform: z.enum(['x_twitter', 'facebook', ...]).default('x_twitter'),
    post_format: z.enum(['text', 'official_statement', ...]).default('text'),
    post_as_page: z.boolean().optional().default(false),
    // <-- shared_article_id and content_flags are MISSING
  }),
});

Required change

const createPostSchema = z.object({
  body: z.object({

Task 2 — Auto-Post News to Social Media (NPC Dual-Publish)

P1

Goal: When the feed engine creates a news article via routeToNews(), it should also create a corresponding social post from the NPC news outlet account. This makes news organically appear on social media as a link post with the article attached.

File to modify
server/services/feedEngineService.ts — routeToNews() function (lines 343–374)

What to add (after the article is inserted and broadcast)

// After article insert + WS broadcast, also create a social post
const outletHandle = '@' + (config.outlet_name || 'newswire')
  .toLowerCase().replace(/[^a-z0-9]/g, '');

const tweetContent = `BREAKING: ${config.headline || inject.title}\n\nFull story: news.sim/${article.id.slice(0, 8)}`;

await supabaseAdmin.from('social_posts').insert({
  session_id: sessionId,
  inject_id: injectId,
  platform: 'x_twitter',
  author_handle: outletHandle,
  author_display_name: config.outlet_name || 'News Wire',
  author_type: 'npc_media',
  content: tweetContent,
  shared_article_id: article.id,
  content_flags: {
    shared_article: {
      id: article.id,
      headline: config.headline || inject.title,
      outlet_name: config.outlet_name || 'News Wire',
      snippet: inject.content.substring(0, 150),
      category: config.category || 'breaking',
    }
  },
  sentiment: 'neutral',
  hashtags: ['#BreakingNews'],
});

// Broadcast to social feed
getWebSocketService().broadcastToSession(sessionId, {
  type: 'social_post.created',
  data: { post: socialPost },
  timestamp: new Date().toISOString(),
});

NewsApp: Track reads

// When setSelectedArticle is called:
function openArticle(article: NewsArticle) {
  setSelectedArticle(article);
  // Fire and forget — track the read
  getAuthHeaders().then(headers =>
    fetch(apiUrl(`/api/social/news/${article.id}/read`), {
      method: 'POST', headers
    })
  );
}
